linux와 표준에 대하여

리눅스는 표준을 잘 지킨 사례라고 할 수 있다.

Multics

  • GE, Dell lab, MIT AI lab 의 multi-Access Computers 프로젝트의 산물
  • 목표: 운영체제의 복잡한 기능울 감추고 시분할, 페이지/세그먼트 메모리 관리, 프로세스 관리, 파일 시스템, 주변장치 관리, 양방향 인터페이스 등등 다양한 기능을 실험하는 프로젝트
  • 프로젝트가 비대해지면서 망하지만 UNIX 개발의 시작이 됨

UNIX

  • Ken Thompson, Ritchie이 게임을 위해 UNIX를 만듬
  • UNIX가 성공하면서 DEC 운영체제가 밀려남
  • 당시 UNIX는 어셈블리어로 작성되어 새로운 CPU가 나오면 거의 새로 작성되어야 했다.
  • Dennis Ritchie는 Brian Kernighan의 도움으로 B 언어를 개량한 C 언어를 개발(1973)
  • UNIX가 C언어로 작성되면서 포팅이 쉬워짐
  • C언어의 부흥으로 어셈블리어의 몰락을 가져옴
  • UNIX는 분열하기 시작
  • BSD UNIX가 탄생 (1975년) 했으나 구심점이 없었다.
  • 기능을 각자 알아서 추가하다 보니 호환성이 사라짐
  • Sysv vs BSD 경쟁 - 명령어도 다름. 운영체제 버전이 다르면 호환이 안됨
  • POSIX가 있어도 더 복잡해진 상황
  • 빌게이츠의 제안으로 SUS - Single UNIX Specification 유닉스 단일 표준안

C Programming Language

  • The C Programming Language (TCPL, K&R)
  • 고급언어이면서도 어셈블리어에 버금가는 성능
  • 낮은 추상화 지원 stdio, File 등
  • 저수준의 하드웨어 조작이 가능
  • 어셈블리어보다 쉬운 언어적 특성
  • ANSI-C(C89): 1989년도 표준
  • C99 - 1999년도 표준
  • C11 : 2011년도 표준

POSIX

  • Portable Oprating system Interface
  • 1988년 POSIX 초안 발표
  • 전기전자공학회(IEEE)의 주도로 system call 표준화 진행
  • POSIX는 API의 문법과 작동에 대한 의미(semantic)만을 담고 있음. 느슨하게 작성됨.
  • 정부 조달의 최소기준

OSF

  • Open Software Foundation 등장
  • 대기업의 반독점을 목적으로 설립된 비영리 단체
  • 1990년 초반 BSD, SVR4, OSF/1 등으로 파편화

X/Open

  • XPG는 manual page의 시초
  • 유닉스 상표권을 소유하게 되나 나중에 Open Group에 넘김
  • SUSv1: XPG 4.2 UNIX95
  • SUSv2: UNIX98 500번
  • SUSv3: SUS 2002 600번 v4는 700

유닉스 이후의 세계

  • GNU에서 토발즈가 만든 Linux kernel을 지원하기 시작
  • Linux는 Monolithic kernel 방식
  • Minix 에서 시작됨 1991 년 linux 발표 1992년 부터 빠른 발전
  • GNU - Gnu is Not Unix
  • GNU 컴파일러 GCC에 의해서 더욱 발전
  • GPL 공개 라이선스를 따름
  • BSD는 4.3 버전을 마지막으로 사라져감
  • Max OSX 2001년 BSD 기반으로 만듬
  • SUS이후 계승되어 표준 대부분을 만족함

Linux 배포판

  • 배포판을 분류하는 기준

    • deb 계열: Debian, Ubuntu, Mint 등
    • rpm 계열: RHEL, CentOS 등
    • RedHat 계열
    • Debian 계열

  • CentOS를 사용하는 경우 EPEL을 사용하는 경우가 많다.

  • debian계열은 패키지 지원이 부족

  • Xfce 경량 데스크탑

  • 센토스는 스타트업등 엔터프라이즈에서 사용

  • 우분투 미려한 데스크탑 환경

  • 페도라는 신기술 및 보안 개발에 사용

  • 임베디드에도 리눅스 사용됨
