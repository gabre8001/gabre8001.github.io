RVIZ로 자동차 움직여 보기

  • 일단 눈으로 보기에 8자와 유사하게 움직이는 것을 해보는데 주안점을 두었다.

자율주행 자동차 기술 소개

자율주행 프로세스

  • 인지

    • 경로탐색, 차량/인프라 통신, 센서
    • 고정지물 인식 및 경로탐색, 변동지물 및 이동물 인식

  • 판단

    • 주행상황 판단 및 주행전략 결정
    • 주행경로 생성

  • 제어

    • 차량제어
    • 엔진 가감속, 조향

요소 기술

  • 고정밀 지도

    • 기존 네비게이션 지도보다 정밀한 지도
    • 100 ~ 200 mm 정밀도 제공
    • 차선정보, 가드레일, 도로 곡률, 결사, 신호등 위치, 교통표식과 같은 부가정보 포함
    • 벡터 맵
    • 포인트 맵

  • Localization 기술

    • 정밀지도와 차량의 센서와 연동하여 차량의 현재 위치 파악

  • Global Path Planning (Route Planning)

    • 목적지가지의 경로 찾기
    • 중간목적지 또는 최종목적지까지의 경로
    • 교차로에서 행동 결정

  • Object Detection

    • 주변 차량, 보행자, 오토바이, 자전거 등을 인식

  • Object Tracking

    • 각 객체에 고유 ID 부여하여 추적, 예상되는 주행경로 예측

  • Local Path Planning (Trajectory Planning)

    • 다음 이동할 곳 경로 찾기(충돌 회피 교려)
    • 여러개의 후보 경로 확보
    • 끊임 없이 후보 삭제, 신규후보 등록 작업을 반복
    • 실시간성이 중요 -> 시스템 최적화 필요

  • Behavior Selector

    • 행위 결정
      • 운전 의사결정
      • 운전 방법, 성향

  • Local Path Following (Trajectory Following)

    • 경로 따라 차량을 운전하기
    • Pure pursuit 알고리즘
      • 차량의 현재 위치를 결정
      • 차향에서 가장 가까운 경로상의 점을 찾는다.
      • 목표점을 찾는다.
      • 곡률을 계산하고 해당 곡률로 차량의 방향을 업데이트 해준다.
      • 차량의 위치를 업데이트 한다.

  • Vehicle Control

    • 주행 제어
      • 원하는 대로 차량을 움직이게 만들기
    • 단순하지 않음
      • 차량 운동학
      • 관성, 마찰력, 미끄러짐 등 있음

ROS 원격 노드 통신

ROS 원격 노드 통신에 대해 알아본다.

ROS 통신

  • 메시지 발행 노드는 자신의 접속 관련 정보를 마스터에게 전달한다.

  • 메시지 구독 노드는 자신의 구독 정보를 마스터에게 알리고 접속할 노드의 정보를 마스터로부터 받는다.

  • 구독 노드와 발행 노드는 직접 연결을 하고 토픽을 주고 받는다.

  • ROS 사용 통신 프로토콜(TCP/IP)

    • XMLRPC: 노드들이 마스터와 통신할 때 이용
    • TCPROS: 각 노드간 통신할 때 이용

RVIZ

RVIZ와 URDF에 대해 알아본다.

RVIZ

  • ROS와 연동하여 사용하는 3D 시각화 툴.
  • RVIZ 구동을 위해서는 아래와 같이 런치 파일 편집을 통해 ROS 실행을 하면 확인 가능하다.
    • pkg를 rviz로, type도 rviz로 생성한 노드를 통해 RVIZ를 실행한다.
    • 3D 시각화를 위해 필요한 파일이 urdf 파일이다.

URDF

  • Unified Robot Description Format

  • 로봇을 구현하는데 쓰이는 하나의 포맷으로 RVIZ에서 시각화를 통해 확인 할 수 있다.

  • 좌표계

    • 위치나 크기를 표현하기 우해 데카르트 좌표계 x,y,z를 사용
    • 회전을 표현하기 위해 오일러 각도 roll, pitch, yaw를 사용
    • 사용 단위
      • 길이: 미터 m
      • 각도: 라디안 radian
      • 질량: 킬로그램 kg
      • 속도: m/s (병진/직진운동)
      • 각속도: radian/s (회전운동)

  • 형상 표현

    • cylinder: 원통
    • box: 상자
    • sphere: 공

  • 색상 표현

    • RGB 3원색과 투과율 A를 0~1사이의 숫자로 정의함

  • 기구의 표현

    • base: 고정 부분(grounded)
    • link: 관절에 연결되는 로봇 팔의 부분
    • joint: 링크를 연결하는 부위로 보통 모터의 회전을 통해 움직임을 만듬

좌표계

  • 위치나 크기를 표현하기 우해 데카르트 좌표계 x,y,z를 사용
  • 회전을 표현하기 위해 오일러 각도 roll, pitch, yaw를 사용

단위

  • 길이: 미터 m
  • 각도: 라디안 radian
  • 질량: 킬로그램 kg
  • 속도: m/s (병진/직진운동)
  • 각속도: radian/s (회전운동)

형상 표현

  • cylinder: 원통
  • box: 상자
  • sphere: 공

색상 표현

  • RGB 3원색과 투과율 A를 0~1사이의 숫자로 정의함

기구의 표현

  • base: 고정 부분(grounded)

  • link: 관절에 연결되는 로봇 팔의 부분

    • visual: 시각화를 위해 형상과 위치를 정의
      • geometry: 형상 정의 원동, 상자, 공
      • origin: 고정축을 기준으로 link 형상의 roll, pitch, yaw 위치를 라디안으로 나타내고 x,y,z 좌표 위치를 미터 단위로 지정
      • material: 현상의 컬러값 지정

  • joint: 링크를 연결하는 부위로 보통 모터의 회전을 통해 움직임을 만듬

    • parent: parent frame의 이름을 지정해서 child frame과 연결
    • child: child frame의 이름을 지정해서 parent frame과 연결
    • origin: 고정축을 기준으로 형상의 r,p,y 위치를 라디안으로 나타내고, x,y,z 좌표 위치를 미터 단위로 지정.
    • joint는 parent의 origin, child는 joint의 origin을 고정축으로 정함
    • limit: joint의 운동범위 제한값을 지정
      • lower: revolution type의 joint에서 최저 각을 지정
      • upper: revolution type의 joint에서 최대 각을 지정
      • effort: N 힘의 최대값을 지정
      • velocity: radian/s 최대 속도를 지정

joint의 동작 정의

  • fixed: 고정
  • revolution: 작동 범위 제한
  • continuous: 연속 회전

ROS 기본

ROS 대해 알아본다.

ROS

  • Robot Operating System

    • 오픈소스 로보 운영체제
    • 소스 공개. 개방형 구조. 활발한 커뮤니티 활동

  • 로봇 소프트웨어르 개발하는 데 필요한 소프트웨어의 집합체

    • 소프트웨어 프레임워크

  • 메타 운영제체(Meta OS), 미들웨어(Middleware)

    • 소프트웨어 모듈 + 라이브러리 집합 + 도구 집합

  • 각종 센서와 모터를 프로그래머가 편하게 사용할 수 있도록 지원

  • 프로토타이핑 작업에 적합

  • 실제 상용 제품에 사용되는 것은 의견이 나뉨

  • 다양하 프로로그래미 언어를 지원 C++ Python

  • 표준화된 ROS 인터페이스를 따르는 HW, SW를 편하게 엮을 수 있음

  • 하드웨어 부품과 소프트웨어 부품으 조립하여 여러 응용 구성 가능

  • 대규모 실행 시스템 및 프로세스에도 적용 가능

  • 다양한 도구 제공

    • RVIZ, RQT, Gazebo, …
    • RVIZ 시각화 도구. 센서데이터를 비롯한 주변환경 변화르 시각화
    • RQT Qt 기반의 GUI 응용 개발 도구. 노드연결 정보를 그래프로 표현. 사용자 상호작용을 UI를 갖춘 응용 개발에 이용
    • GAZEBO 물리 엔진 기반의 3차원 시뮬레이터. 시뮬레이터 제작 및 모델링에 이용

  • 다양한 OS 환경에서 통일된 방법으로 상호작용을 구현하는 것이 가능

    • Linux, OS X, Windows, Rasbian, QNX, RTOS, Android, IOS 등
    • 표준화된 통신 프로토콜으 따르는 이기종간의 메시지 교환이 가능

  • ROS 버전

    • ROS Melodic Morenia 2018
    • ROS Linar Loggerhead 2017
    • ROS Kinetic Kame 2016
    • ROS Indigo Igloo 2014

  • ROS 통신

    • 토픽의 발행/구동 (publish & subscribe) 기반의 메시지 교환
    • 이기종간의 메시지 교환 가능

  • ROS 핵심 기능

    • 노드간 통신을 기반으로 전체 시스템을 구동시킴
    • 하드웨어 부품 또는 소프트웨어 모듈에 노드가 하나씩 할당됨
    • 네트워크를 통한 원격 연동도 가능

  • ROS 기본 용어

    • 마스터
      • 서로 다른 노드들 사이의 통신을 총괄 관리
      • ROS Core 라고 부름
    • 노드 Node
      • 실행가능한 최소 단위. 프로세스로 이해 가능
      • ROS에서 발생하는 통신의 주체
      • HW장치에 대해 하나씩의 노드, SW 모듈에 대해 하나씩의 노드
    • 토픽 Topic
      • 노드들이 관심을 가지고 있는 이야깃거리
      • 그 안에 들어있는 실제 데이터를 메시지라 부름
      • 센서데이터, 카메라 이미지, 액츄에이터 제어 명령 등…
    • 발행자 Publishers
      • 특정 토픽에 메시지를 담아 외부로 송신하는 노드
      • 센서, 카메라, 모터제어 알고리즘
    • 구독자 Subscribers
      • 특정 토픽에 담겨진 메시지를 수신하는 노드
      • 엑츄에이터 제어기, 데이터 시각화 도구
    • 패키지 Packages
      • 하나 이상의 노드와 노드의 실행을 위한 정보 등을 묶어 놓은 단위
      • 노드, 라이브러리, 데이터, 파라미터 등을 포함

  • ROS 노드간 통신 기본 과정

    • 통신이 이루어지기 전에 통신을 원하는 노드는 마스터에 의뢰
    • 연결해야 하는 노드의 정보(주소)를 얻어오고, 서로 접속 정보를 교환
    • 통신환경 구축이 완료 되면
    • 노드간 통신은 마스터를 거치지 않고 직접 이루어짐

  • ROS 노드간 통신의 두 가지 방식

    • 토픽 방식의 통신
      • 일방적이고 지속적인 메시지 전송
      • 1:1 및 1:N, N:N 통신도 가능
    • 서비스 방식의 통신
      • 서버가 제공하는 서비스에 클라이언트가 요청을 보내고 응답을 받는 방식
      • 양방향 통신, 일회성 메시지 송수신

  • ROS 노드간 통신 시나리오

    • 마스터 시동 - 통신이 이루어지려면 ROS core가 실행되고 있어야 함. 마스터의 정보는 모든 노드가 알 수 있음
    • 구독자 노드 구동 - 특정 토픽에 발행 되는 메시지를 수신하기를 요청. 자신의 모든 정보를 마스터에게 알림
    • 발행자 노드 구동 - 특정 토픽 메기지를 발행하겠다는 의사 전달. 역시 자신의 모든 정보를 마스터에게 알림
    • 노드 정보 전달 - 마스터가 발행자 정보를 구독자에게 전달
    • 노드간 접속 요청 - 구독자 노드가 발행자 노드에 TCP ROS 접속을 요청
    • 노드간 접속 요청에 대한 응답 - 발행자 노드가 자신의 TCPROS URI(포트 정보 포함)를 전송하여 응답
    • TCPROS 접속 - 발행자 노드와 구독자 노드 사이에 소켓 연결이 이루어짐
    • 메시지 전송 - 발행자 노드가 구독자 노드에게 메시지 전송(토픽)
    • 메시지 전송 반복 - 접속이 한번 이루어진 뒤에는 별도 절차 없이 지속적으로 메시지 송수신

roscore | master + rosout + parameter server
rosrun | 패키지 노드 실행 rosrun [package name][node_name]
roslaunch | 패키지 노드를 여러 개 실행 rosluanch [package_name][file.launch]
rosclean | ros 로그파일 검사 및 삭제

rostopic | 토픽 정보 확인 rostopic info /imu

rosparam | 파라미터 정보 확인, 수정
rosbag | 메세지 기록, 재생
rosmsg | 메세지 정보 확인
rosversion| 패키지 및 배포 버전정보 확인
roswtf | ros 시스템 검사

catkin_create_pkg | catkin 빌드 시스템으로 패키지 생성
catkin_make | catkin 빌드 시스템으로 빌드
catkin_eclipse | 패키지를 eclipse에서 사용할 수 있게 변경
catkin_prepare_release | changelog 정리 및 버전 태깅
catkin_init_workspace | catkin 빌드 시스템의 작업 폴더 초기화
catkin_find | 검색

  • ROS package 명령어

rospack | 패키지와 관련된 정보 보기
rosinstall | 추가 패키지 설치
rosdep | 해당 패키지의 의존성 파일 설치
roslocate | 패키지 정보 관련 명령어
rosmake | 패키지 빌드(구 시스템에서 사용)
roscreate-pkg | 패키지 자동 생성(구 시스템에서 사용)
