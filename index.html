Array의 Broadcasting

  • 피연산자가 연산 가능하도록 변환을 하여 연산을 진행하는 것을 Broadcasting 이라 함.

  • M x N, M x 1

  • M x 1, 1 x N

영벡터(행렬)

  • 원소가 모두 0인 벡터(행렬)

일벡터(행렬)

  • 원소가 모두 1인 벡터(행렬)

대각행렬(diagonal matrix)

  • Main diagonal을 제외한 성분이 0인 행렬

항등행렬(identity matrix)

  • Main diagonal이 1인 대각행렬
  • 인자로 들어가는 dtype의 기본 값은 float

행렬곱(dot product)

  • 행렬간의 곱연산

트레이스(trace)

  • main diagonal의 합

행렬식(determinant)

  • 행렬을 대표하는 값들 중 하나

역행렬(inverse matrix)

  • 행렬 A에 대해 AB = BA = I 를 만족하는 행렬 B = A^-1

고유값과 고유벡터(eigenvalue and eigenvector)

Mat

  • opencv에서 사용되는 이미지를 담는 클래스
  • 어마무시하게 많이 사용되는 클래스
  • opencv에서 컬러는 CV_8UC3 으로 나타내고, 흑백은 CV_8UC1로 나타낸다.
    • 8은 8비트 즉 0~255 값으로 색정보를 담는다.
    • UC는 unsigned char를 나타내고,
    • 3 혹은 1은 8비트짜리가 3개 혹은 1개 있다는 것을 의미한다.
    • color는 BGR 세가지 정보를 위해 8비트가 세개 있어야 하고, gray scale은 8비트가 하나만 있어도 표현이 가능하다.

  • Mat 클래스는 이미지를 담다보니 이차원 배열과 유사한 연산이 가능하다.

Open CV

open cv 설치에 대해 알아본다. 본문에서는 윈도우10과 비주얼 스투디오 2022를 사용하는 환경을 기준으로 한다.

설치

  • 설치 파일 다운로드

    • https://opencv.org/releases 에서 파일 다운로드
    • 이 글에서는 opencv 버전 4.5.5를 기준으로 한다.
    • opencv-4.5.5vc14_vc15.exe 파일 다운로드한다.

  • 설치 실행 파일 이용

    • opencv-4.5.5vc14_vc15.exe 파일을 실행하면 압축이 풀린다.
    • 압축을 풀 때 위치를 지정할 수 있다.
    • 설치는 압축을 푼것으로 완료.

  • opencv 파일 설명

    • opencv/build/include/opencv2 경로에 헤더파일들이 위치한다.
    • opencv/build/x64 아래 vc14 vc15 디렉토리가 있다.
    • vc14는 visual studio 2015 버전을 이용해 빌드한 파일들이 있고, vc15는 2017버전을 이용해 빌드한 파일들이 있다.
    • visual studio 2022 버전에서는 vc15를 사용한다.
    • vc15/bin 경로 아래에는 dll 파일이 있다.
    • vc15/lib 경로 아래에는 lib 파일이 있다.

  • 환경변수 등록

    • 설치한 opencv 를 사용하기 위해 환경 변수 등록을 해준다.
      • 변수 이름 OPENCV_DIR
      • 변수 값 opencv/build
    • path 항목 편집
      • %OPENCV_DIR%/x64/vc15/bin 항목 추가
    • cmd 에서 설치 확인
      • opencv_version 명령어를 입력하여 버전 정보를 확인 할 수 있으면 설치 완료

Visual Studio 2022 프로젝트 만들기

자율주행 자동차 기술 소개

자율주행 프로세스

  • 인지

    • 경로탐색, 차량/인프라 통신, 센서
    • 고정지물 인식 및 경로탐색, 변동지물 및 이동물 인식

  • 판단

    • 주행상황 판단 및 주행전략 결정
    • 주행경로 생성

  • 제어

    • 차량제어
    • 엔진 가감속, 조향

요소 기술

  • 고정밀 지도

    • 기존 네비게이션 지도보다 정밀한 지도
    • 100 ~ 200 mm 정밀도 제공
    • 차선정보, 가드레일, 도로 곡률, 결사, 신호등 위치, 교통표식과 같은 부가정보 포함
    • 벡터 맵
    • 포인트 맵

  • Localization 기술

    • 정밀지도와 차량의 센서와 연동하여 차량의 현재 위치 파악

  • Global Path Planning (Route Planning)

    • 목적지가지의 경로 찾기
    • 중간목적지 또는 최종목적지까지의 경로
    • 교차로에서 행동 결정

  • Object Detection

    • 주변 차량, 보행자, 오토바이, 자전거 등을 인식

  • Object Tracking

    • 각 객체에 고유 ID 부여하여 추적, 예상되는 주행경로 예측

  • Local Path Planning (Trajectory Planning)

    • 다음 이동할 곳 경로 찾기(충돌 회피 교려)
    • 여러개의 후보 경로 확보
    • 끊임 없이 후보 삭제, 신규후보 등록 작업을 반복
    • 실시간성이 중요 -> 시스템 최적화 필요

  • Behavior Selector

    • 행위 결정
      • 운전 의사결정
      • 운전 방법, 성향

  • Local Path Following (Trajectory Following)

    • 경로 따라 차량을 운전하기
    • Pure pursuit 알고리즘
      • 차량의 현재 위치를 결정
      • 차향에서 가장 가까운 경로상의 점을 찾는다.
      • 목표점을 찾는다.
      • 곡률을 계산하고 해당 곡률로 차량의 방향을 업데이트 해준다.
      • 차량의 위치를 업데이트 한다.

  • Vehicle Control

    • 주행 제어
      • 원하는 대로 차량을 움직이게 만들기
    • 단순하지 않음
      • 차량 운동학
      • 관성, 마찰력, 미끄러짐 등 있음
