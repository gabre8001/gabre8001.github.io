Mat

  • opencv에서 사용되는 이미지를 담는 클래스
  • 어마무시하게 많이 사용되는 클래스
  • opencv에서 컬러는 CV_8UC3 으로 나타내고, 흑백은 CV_8UC1로 나타낸다.
    • 8은 8비트 즉 0~255 값으로 색정보를 담는다.
    • UC는 unsigned char를 나타내고,
    • 3 혹은 1은 8비트짜리가 3개 혹은 1개 있다는 것을 의미한다.
    • color는 BGR 세가지 정보를 위해 8비트가 세개 있어야 하고, gray scale은 8비트가 하나만 있어도 표현이 가능하다.

Open CV

open cv 설치에 대해 알아본다. 본문에서는 윈도우10과 비주얼 스투디오 2022를 사용하는 환경을 기준으로 한다.

설치

  • 설치 파일 다운로드

    • https://opencv.org/releases 에서 파일 다운로드
    • 이 글에서는 opencv 버전 4.5.5를 기준으로 한다.
    • opencv-4.5.5vc14_vc15.exe 파일 다운로드한다.

  • 설치 실행 파일 이용

    • opencv-4.5.5vc14_vc15.exe 파일을 실행하면 압축이 풀린다.
    • 압축을 풀 때 위치를 지정할 수 있다.
    • 설치는 압축을 푼것으로 완료.

  • opencv 파일 설명

    • opencv/build/include/opencv2 경로에 헤더파일들이 위치한다.
    • opencv/build/x64 아래 vc14 vc15 디렉토리가 있다.
    • vc14는 visual studio 2015 버전을 이용해 빌드한 파일들이 있고, vc15는 2017버전을 이용해 빌드한 파일들이 있다.
    • visual studio 2022 버전에서는 vc15를 사용한다.
    • vc15/bin 경로 아래에는 dll 파일이 있다.
    • vc15/lib 경로 아래에는 lib 파일이 있다.

  • 환경변수 등록

    • 설치한 opencv 를 사용하기 위해 환경 변수 등록을 해준다.
      • 변수 이름 OPENCV_DIR
      • 변수 값 opencv/build
    • path 항목 편집
      • %OPENCV_DIR%/x64/vc15/bin 항목 추가
    • cmd 에서 설치 확인
      • opencv_version 명령어를 입력하여 버전 정보를 확인 할 수 있으면 설치 완료

자율주행 자동차 기술 소개

자율주행 프로세스

  • 인지

    • 경로탐색, 차량/인프라 통신, 센서
    • 고정지물 인식 및 경로탐색, 변동지물 및 이동물 인식

  • 판단

    • 주행상황 판단 및 주행전략 결정
    • 주행경로 생성

  • 제어

    • 차량제어
    • 엔진 가감속, 조향

요소 기술

  • 고정밀 지도

    • 기존 네비게이션 지도보다 정밀한 지도
    • 100 ~ 200 mm 정밀도 제공
    • 차선정보, 가드레일, 도로 곡률, 결사, 신호등 위치, 교통표식과 같은 부가정보 포함
    • 벡터 맵
    • 포인트 맵

  • Localization 기술

    • 정밀지도와 차량의 센서와 연동하여 차량의 현재 위치 파악

  • Global Path Planning (Route Planning)

    • 목적지가지의 경로 찾기
    • 중간목적지 또는 최종목적지까지의 경로
    • 교차로에서 행동 결정

  • Object Detection

    • 주변 차량, 보행자, 오토바이, 자전거 등을 인식

  • Object Tracking

    • 각 객체에 고유 ID 부여하여 추적, 예상되는 주행경로 예측

  • Local Path Planning (Trajectory Planning)

    • 다음 이동할 곳 경로 찾기(충돌 회피 교려)
    • 여러개의 후보 경로 확보
    • 끊임 없이 후보 삭제, 신규후보 등록 작업을 반복
    • 실시간성이 중요 -> 시스템 최적화 필요

  • Behavior Selector

    • 행위 결정
      • 운전 의사결정
      • 운전 방법, 성향

  • Local Path Following (Trajectory Following)

    • 경로 따라 차량을 운전하기
    • Pure pursuit 알고리즘
      • 차량의 현재 위치를 결정
      • 차향에서 가장 가까운 경로상의 점을 찾는다.
      • 목표점을 찾는다.
      • 곡률을 계산하고 해당 곡률로 차량의 방향을 업데이트 해준다.
      • 차량의 위치를 업데이트 한다.

  • Vehicle Control

    • 주행 제어
      • 원하는 대로 차량을 움직이게 만들기
    • 단순하지 않음
      • 차량 운동학
      • 관성, 마찰력, 미끄러짐 등 있음

RVIZ

RVIZ와 URDF에 대해 알아본다.

RVIZ

  • ROS와 연동하여 사용하는 3D 시각화 툴.
  • RVIZ 구동을 위해서는 아래와 같이 런치 파일 편집을 통해 ROS 실행을 하면 확인 가능하다.
    • pkg를 rviz로, type도 rviz로 생성한 노드를 통해 RVIZ를 실행한다.
    • 3D 시각화를 위해 필요한 파일이 urdf 파일이다.

URDF

  • Unified Robot Description Format

  • 로봇을 구현하는데 쓰이는 하나의 포맷으로 RVIZ에서 시각화를 통해 확인 할 수 있다.

  • 좌표계

    • 위치나 크기를 표현하기 우해 데카르트 좌표계 x,y,z를 사용
    • 회전을 표현하기 위해 오일러 각도 roll, pitch, yaw를 사용
    • 사용 단위
      • 길이: 미터 m
      • 각도: 라디안 radian
      • 질량: 킬로그램 kg
      • 속도: m/s (병진/직진운동)
      • 각속도: radian/s (회전운동)

  • 형상 표현

    • cylinder: 원통
    • box: 상자
    • sphere: 공

  • 색상 표현

    • RGB 3원색과 투과율 A를 0~1사이의 숫자로 정의함

  • 기구의 표현

    • base: 고정 부분(grounded)
    • link: 관절에 연결되는 로봇 팔의 부분
    • joint: 링크를 연결하는 부위로 보통 모터의 회전을 통해 움직임을 만듬

좌표계

  • 위치나 크기를 표현하기 우해 데카르트 좌표계 x,y,z를 사용
  • 회전을 표현하기 위해 오일러 각도 roll, pitch, yaw를 사용

단위

  • 길이: 미터 m
  • 각도: 라디안 radian
  • 질량: 킬로그램 kg
  • 속도: m/s (병진/직진운동)
  • 각속도: radian/s (회전운동)

형상 표현

  • cylinder: 원통
  • box: 상자
  • sphere: 공

색상 표현

  • RGB 3원색과 투과율 A를 0~1사이의 숫자로 정의함

기구의 표현

  • base: 고정 부분(grounded)

  • link: 관절에 연결되는 로봇 팔의 부분

    • visual: 시각화를 위해 형상과 위치를 정의
      • geometry: 형상 정의 원동, 상자, 공
      • origin: 고정축을 기준으로 link 형상의 roll, pitch, yaw 위치를 라디안으로 나타내고 x,y,z 좌표 위치를 미터 단위로 지정
      • material: 현상의 컬러값 지정

  • joint: 링크를 연결하는 부위로 보통 모터의 회전을 통해 움직임을 만듬

    • parent: parent frame의 이름을 지정해서 child frame과 연결
    • child: child frame의 이름을 지정해서 parent frame과 연결
    • origin: 고정축을 기준으로 형상의 r,p,y 위치를 라디안으로 나타내고, x,y,z 좌표 위치를 미터 단위로 지정.
    • joint는 parent의 origin, child는 joint의 origin을 고정축으로 정함
    • limit: joint의 운동범위 제한값을 지정
      • lower: revolution type의 joint에서 최저 각을 지정
      • upper: revolution type의 joint에서 최대 각을 지정
      • effort: N 힘의 최대값을 지정
      • velocity: radian/s 최대 속도를 지정

joint의 동작 정의

  • fixed: 고정
  • revolution: 작동 범위 제한
  • continuous: 연속 회전

      • 특정 토픽에 담겨진 메시지를 수신하는 노드

  • ROS 노드간 통신 기본 과정

    • 통신이 이루어지기 전에 통신을 원하는 노드는 마스터에 의뢰
    • 연결해야 하는 노드의 정보(주소)를 얻어오고, 서로 접속 정보를 교환
    • 통신환경 구축이 완료 되면
    • 노드간 통신은 마스터를 거치지 않고 직접 이루어짐

  • ROS 노드간 통신의 두 가지 방식

    • 토픽 방식의 통신
      • 일방적이고 지속적인 메시지 전송
      • 1:1 및 1:N, N:N 통신도 가능
    • 서비스 방식의 통신
      • 서버가 제공하는 서비스에 클라이언트가 요청을 보내고 응답을 받는 방식
      • 양방향 통신, 일회성 메시지 송수신

  • ROS 노드간 통신 시나리오

    • 마스터 시동 - 통신이 이루어지려면 ROS core가 실행되고 있어야 함. 마스터의 정보는 모든 노드가 알 수 있음
    • 구독자 노드 구동 - 특정 토픽에 발행 되는 메시지를 수신하기를 요청. 자신의 모든 정보를 마스터에게 알림
    • 발행자 노드 구동 - 특정 토픽 메기지를 발행하겠다는 의사 전달. 역시 자신의 모든 정보를 마스터에게 알림
    • 노드 정보 전달 - 마스터가 발행자 정보를 구독자에게 전달
    • 노드간 접속 요청 - 구독자 노드가 발행자 노드에 TCP ROS 접속을 요청
    • 노드간 접속 요청에 대한 응답 - 발행자 노드가 자신의 TCPROS URI(포트 정보 포함)를 전송하여 응답
    • TCPROS 접속 - 발행자 노드와 구독자 노드 사이에 소켓 연결이 이루어짐
    • 메시지 전송 - 발행자 노드가 구독자 노드에게 메시지 전송(토픽)
    • 메시지 전송 반복 - 접속이 한번 이루어진 뒤에는 별도 절차 없이 지속적으로 메시지 송수신

bash shell 작동원리

  • 셸에서 실행되는 UNIX 명령어는 sub-shell 이 생성되어 실행

  • 서브셸이란 fork-exec로 만들어지는 child process 셸

  • 부모 프로세스는 자식 프로세스의 exit code 를 읽어햐한다.

  • 자식이 성공했는지 실패했는지 알수 있다. 0이면 성공 아니면 실패

  • zombie

    • 부모 프로세스가 종료된 자식 프로세스의 exit status르 처리하기 전의 상태
    • 자식 프로세스를 zombie process 혹은 defunct process 라고 부름
    • 모든 프로세스의 종료는 zombie 상태를 거쳐서 해소되기 때문에 프로세스가 종료되는 정상적인 과정중에 하나.
    • 좀비 프로세스가 생긴다는 건 부모 프로세스에 문제가 있다는 것!

  • echo $? 이전 명령어의 리턴값

  • rm --force 옵션에 대하 오래

      • IPv4 32bit 주소 체계. 8bit씩 끊어서 xxx.ooo.xxx.ooo
      • IPv6 128bit 주소 체계. 무지 많다.
      • IPv4-mapped IPv6 ::ffff:IPv4_addr
      • dual stack IPv4와 IPv6를 혼용해서 사용하기 위한 설계. 대부분 OS 지원 (win xp 제외)
      • A,B,C 클래스 뭐시기는 없어짐
      • CIDR(Classless Inter-Domain Routing) IP클래스와 상관럾이 서브넷을 지정하여 자르는 것을 의미. 서브넷 마스크 비트 크기.
      • public IP 공인주소(인터넷에서 유일한 주소), private IP(인터넷에 직접 연결되지 않은 주소)
    • NIC: Network Interface Card == 랜카드
    • Wired Network (Wired connection): 유선 네트워크(유선 연결)
    • Wireless Network (Wireless connection): 무선 네트워크(무선 연결)
    • LAN: Local Area Network
    • WAN: Wide Area Network
    • SELinux(Security Enhanced Linux) 커널 레벨에서의 중앙 집중식 보안 기능
    • SELinux 보안 레벨
      • enforcing(강제): 보안설정에 걸리면 강제로 막음(많은 보안 지식필요)
      • permissive(허가): 보안설정에 걸리면 허용하되 로그 출력(실습시 사용 권장)
      • disable(비활성): SELinux 사용하지 않음

File System

  • FS는 OS의 매우 큰 부분을 차지한다. read, wirte 작업을 수행

  • FS는 넓게 보면 데이터 베이스의 일종. 파일을 저장하는 데이터 베이스

  • 종류

    • ext4: 리눅스 고유 파일 시스템
    • ext3, ext2: ext4의 이전 버전
    • xfs: 실리콘 그래픽스의 저널링 파일 시스템(RHEL7부터 기본 파일시스템)

    • Btrfs: zfs와 비슷한 컨셉으로 개발된
    • ntfs: 윈도우즈 NT 계열에서 사용하는 파일 시스템
    • fat32: FAT 32비트 버전
    • exfat: FAT 32비트 확장 버전

    • partitioning: fdisk, parted
    • file system: mkfs, fsck or xfs_*
    • mount: mount/unmount. /etc/fstab

  • Partition

    • physical/logical 구획이 있다. 현대 OS는 logical partition을 선호
    • logical portition을 사용하면 물리적 한계를 가진 디스크를 묶어서 보다 큰 용량의 파티션을 만들 수 있다. logical volume(LVM)
    • physical disklabel type
      • DOS 방식: 2TB 제한, fdisk 사용
      • GPT 방식: 무제한 용량, parted/gparted for gdisk 사용
    • Dos partition 종류
      • primary partition(주 파티션) 하나의 디스크에 4개 까지 만들 수 있음
      • extended partition(확장 파티션) 4개 이상 파티션이 필요할 때 primary partition 대신 1개 만들 수 있음. extended partition은 여러 개의 logical drive로 나눌 수 있음.

      • /dev/sd SCSI disk, serial type SATA, USB

      • 저성능 I/O에서 효율이 높다(IoT 기기)
      • 삭제된 파일 복구 가능
    • XFS
      • 저널링 기반 대용량 파일 시스템
      • SGI 실리콘 그래픽스사에서 개발 IRIX 유닉스에서 사용. 후 오픈소스로 이전
      • 2000년 초반에 포팅되어 많은 기간동안 테스트됨. 대부분의 고성능 서버형 리눅스 배포판의 기본 파일 시스템으로 사용됨. Debian, RHEL7

      • 대용량 파일 처리시 성능이 좋음
      • kernel service(xfs*)가 관리
      • 삭제된 파일을 복구 할 수 없다.
      • 깨진 파일 시스템 복구를 위한 명령어로 xfs_repair를 제공
    • mkfs
      • make file system 명령어
      • mkfs [-t fstype] [option] <device>
      • lsblk 추가된 disk 확인 인식이 안되있으면
      • ls /sys/class/scsi_host/ | while read hostdev ; do echo “- - -” /sys/class/scsi_host/$hostdev/scan ; done 입력 후

vim 시작

  • vim [filename] 으로 실행
  • find . -name “*.txt” | vim -
  • vi의 작동모드는 일반모드(normal mode), 입력모드(insert mode), 명령행모드(command-line mode), 비주얼모드(visual mode)가 있음
    • normal mode에서 a,i,o 입력 -> insert mode 반대는 ESC 입력
    • normal mode에서 : 입력 -> command mode 반대는 ESC, 실행
    • 노말모드에서 단축키로 실행
    • a: append는 현재 커서 위치에서 한칸 뒤로 이동한 후 입력 모드로 전환 A는 행의 가장 마지막
    • i: insert는 현재 커서 위치에서 입력 모드로 전환 I는 행의 맨 앞
    • o: 커서 위치 아래에 행이 생기면서 입력모드로 전환
    • !: force 명령으로 강제 시행을 의미
  • 커서 이동
    • 일반모드에서 화살표키 대신에 h(left), j(down), k(up), l(right) 을 이용하여 커서를 이동
    • ^: 행의 맨 앞으로 이동(caret)
    • $: 행의 맨 끝으로 이동
    • <CTRL-B>: page up backward
    • <CTRL-F>: page down forward
    • <CTRL-U>: 1/2 page up upward
    • <CTRL-D>: 1/2 page down downward
    • :# -> #행으로 이동
    • #gg: #행으로 이동 # 생략 시 맨위로 이동
    • #G: #행으로 이동 # 생략 시 마지막으로 이동

buffer 사용

    • x: 커서에 위치한 문자 삭제 = delete키
    • dd(:d): 현재 행을 삭제 (3dd는 세줄 삭제)

    • yy, :y, Y: 현재 행을 레지스터에 복사(yank)
  • u: (undo) 바로 이전에 행한 명령 한 개를 취소

range

  • 명령행 모드에서 범위를 지정해서 명령
  • :20d - 20번 행을 삭제
  • :10,25d - 10~25번 행을 삭제
  • :10,$d - 10~마지막 행까지 삭제
  • :&y - 문서 전체를 복사. %는 1,$와 동일
  • :.,+20y - 현재 행부터 아래로 스무행을 복사
  • :-10,+5d - 현재 행부터 위로 10행, 아래로 5행, 총 열여섯 행을 삭제
  • :40pu - 40번 행에 레지스터의 내용 붙여넣기
  • 범위 연산 meta character
    • .: 현재 행을 의미(relative path의 현재 디렉터리와 유사)
    • $: 마지막 행을 의미 (REGEX의 $와 유사)
    • +#: 현재 위치에서 #만큼 아래 행을 의미

    • %: 문서(파일) 전체를 의미

    • *: 현재 커서에 위치한 단어를 전방탐색함
    • #: 현재 커서에 위치한 단어를 후방탐색함
    • n: 다음 탐색 결과를 찾음(전방탐색의 경우 전방으로, 후방탐색의 경우 후방으로 이동)
    • N: n과 반대방향으로 다음 탐색
    • %: 괄호의 짝을 찾아줌
    • case-sensitive searching
      • \c 접두어를 추가할 경우 e.g. /\cinter 로 실행하면 inter의 대소문자 무시 \C(대문자c)는 반대 기능
      • :set ignorecase 는 ignorecase 활성화 반대는 :set noignorecase
    • /[a-g]re: are, bre,cre ~ gre 를 모두 찾음. REGEX 문법을 따름.
    • :nohls - :set nohls 검색 후 밝게 표시된 단어 옵션을 해제
    • \M magic이라 불린다. 해석을 금지하는 접두어. \M[section] 이렇게 하면 REGEX로 해석을 안하고 그냥 문자열로 인식
  • 문자열 교체
    • sed의 기능이 import 된 것. sed 문법과 동일
    • :[range]s/<찾는 문자열>/<교체할 문자열>/<옵션> separator로 slash가 주로 사용되지만 다른 문자를 사용해도 무방(예: 콤마)
    • 명령어 옵션
      • g: (global) 검색된 문자열 모두를 교체. 이 옵션 생략시 처음 1개만 교체
      • i: (ignore case) 대소문자 무시
      • c: (confirm) 교체 할 때마다 확인
      • e: 교체 과정 중 에러 무시
    • 찾으려는 문자열에 /가 들어있을 경우 escape문자를 추가한다.
      • :%s//home/linux//home/unix/g 를 아래 처럼 써도 무방하다.
      • :%s,/home/linux,/home/unix,g
    • 특수 문자의 교체
      • 개행문자 DOS(CR+LF) UNIX(LF)
      • CR 입력: ^V^M <CTRL-V> 대신 <CTRL-Q> 사용 가능
      • :%s/CR입력//g 혹은 :%s/\r//g
      • 실전에서 :set ff=dos 혹은 :set ff=unix 로 설정 후 저장하는 방법 씀
    • ASCII를 코드값으로 입력
      • <CTRL-V> ###: ###에 10진수를 사용하여 ASCII 입력
      • <CTRL-V> o###: ###에 8진수를 사용하여 ASCII 입력
      • <CTRL-V> x###: ###에 16진수를 사용하여 ASCII 입력
      • man ascii 로 아스키코드 확인 가능
    • 숫자 증감: 숫자 위에서 <CTRL-A> 누르면 증가 <CTRL-X> 누르면 감소

파일 인코딩

  • fileencodings (fencs)
    • 파일을 읽을 때 확인할 encoding list를 설정. .vimrc 에 list로 설정. 순서 중요
    • set fencs=value e.g. fencs=usc-bom,utf-8,korea,latin-1
    • BOM(Byte Order Mark) UCS(Universal coded Charater Set) 주로 생략하나, 관습적으로 옵션 설정시 앞부분에 넣어줌
    • 사용 가능 문자 셋 :help encoding-values
      • utf-8, utf8, usc-bom, korea(이건 별칭으로 유닉스에서는 euc-kr로 윈도에서는 cp949로 바뀜), euc-kr, cp949, japan, latin1, ansi(영문 아스키 방식)
    • :set fenc=korea 로 하면 현재 사용 문자 인코딩을 설정하는 것
    • linux to window 파일을 가져갈 때 UTF-8 을 지원하는 에디터가 아닐 경우 변환을 거쳐야함.
      • :set ff=dos :set fenc=korea :wq 명령어를 통해 변환 :sav newfilename.txt 로 다른이름 저장
    • window to linux 파일을 가져올 때 fencs가 잘 설정되어있으면 자동 변환됨. 이후 저장은 dos 형태로 되기 때문에
      • :set ff=unix :set fenc=utf8 :wq 명령어를 통해 리눅스에서 추후 사용시 편하도록 변환 :sav 로 다른 이름 저장

    • % 가장 가까운 괄호 짝으로 이동
    • (, ) 문장 단위의 시작 위치, 끝 위치로 이동
    • {, } 문단 단위의 시작 위치, 끝 위치로 이동
    • [[,]] 블록 단위의 시작 위치, 끝 위치로 이동

  • abbreviation

    • 특정 단어 입력시 대체 입력하는 기능
    • "내멜"을 입력하면 실제 이메일이 입력되는 기능
    • ia는 insert mode 에서만 작동. ca는 command mode 에서만 작동
    • ab 내맬 abcdvvv@gmail.com
    • ab projectsrc /export/2/local/pjr2/mds/src
    • ia 시간0 <C-R>=strftime(“%Y.%m.%d-%H:%M:%S”)<CR>
    • ia tlrks1 <C-R>=strftime(“%c”)<CR>
    • ca 기능을 이용하면 한글상태 오타를 변환할 수 있다.
      • ca ㅈ w
      • ca ㅈㅂ wq
      • ca ㅌ x
      • .vimrc 파일이 EUC-KR로 되어있으면 UTF-8 무서 편집시 작동하지 않을 수 있다. fenc=euc-kr은 안됨!
        명령 설명
        :ab [lhs] 현재 설정된 모든 약어 목록을 출력. lhs에 약어 지정시 해당 약어 정보만 출력
        :ab {lhs} {rhs} 약어 lhs를 rhs 로 변환
        :unab {lhs} 약어 lhs 해제
        :abclear 설정된 모든 약어 해제
        :ia {lhs} {rhs} ab와 기능은 같지만 입력 모드에서만 작동
        :ca {lhs} {rhs} ab와 기능은 같지만 명령행 모드에서만 작동
