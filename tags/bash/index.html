Bash 배경

  • Shell 이란

    • application을 실행시키는 도구
    • shell 에서 내리는 명령어는 해석된 뒤에 자시 프로세스로 만들어져 실행됨
    • Bourne shell /bin/sh 표준 셸 POSIX
      • Stephen R. Bourne이 작성 1977
      • 기본 유닉스 셸
      • 기능이 미약. 대부분의 기능이 외부 명령어로 대체
    • C shell /bin/csh
      • Berkly의 Bill Joy가 작성. BSD UNIX에 포팅
      • C 언어와 비슷하 문법으로 C shell 이라 명명
      • sh와 너무 달라 외면. 후에 개량형으로 TC sh 등장
    • Korn shell /bin/ksh
      • Bell Lab의 David Korn이 작성 1983
      • csh 장점과 bourne shell 기능을 모두 포함
      • vi prompt mode 지원
      • 다양한 변수형, 사칙 연산 기능 확장
      • 현대적인 상용 유닉스 기본셸
    • bash /bin/bash 가장 유명하 셸
      • Bourne again shell. born again sh
      • GNU에서 발전시킨 개량판 1989
      • 현대적인 셸의 기능 대부분이 추가되어 Korn shell과 호환
      • Korn shell을 벤치마킹
      • build in cmd로 대체 = 성능 향상
    • Zsh /bin/zsh
      • Paul Falstad 작성 1990
      • Oh My ZSH! 등으로 커스터마이징 하느 경우 많음
      • 퍼포먼스 문제가 존재
      • Mac OSX catalina 2019 bash에서 zsh 로 변경

bash shell 작동원리

  • 셸에서 실행되는 UNIX 명령어는 sub-shell 이 생성되어 실행

  • 서브셸이란 fork-exec로 만들어지는 child process 셸

  • 부모 프로세스는 자식 프로세스의 exit code 를 읽어햐한다.

  • 자식이 성공했는지 실패했는지 알수 있다. 0이면 성공 아니면 실패

  • zombie

    • 부모 프로세스가 종료된 자식 프로세스의 exit status르 처리하기 전의 상태
    • 자식 프로세스를 zombie process 혹은 defunct process 라고 부름
    • 모든 프로세스의 종료는 zombie 상태를 거쳐서 해소되기 때문에 프로세스가 종료되는 정상적인 과정중에 하나.
    • 좀비 프로세스가 생긴다는 건 부모 프로세스에 문제가 있다는 것!

  • echo $? 이전 명령어의 리턴값

  • rm --force 옵션에 대하 오래
    • UNIX command 특히 rm과 같으 optional하 기능에 --force 옵션이 존재하는 것은 child process로 작동할 때 exit code르 조작하기 위함.

  • make 외에 gradle이나 다른 빌드 툴도 동일한 기능으로 작동하느 것들이 많다.

  • 프로세스의 main 함수의 리턴값 == exit code

  • 부모 프로세스가 자식 프로세스가 어떻게 종료되었는지 알수 있다.

  • shell script도 정상 작동이 아닌 경우 return 값을 0이 아닌 값을 줘야한다.

  • 셸의 명령 기능을 조합하여 batch 처리를 목표로 한다.

    • 외부 명령어와 built-in 명령어르 조합
    • 그 외에 perl(정규표현식 사용 많을 때), python(c언어와의 연동을 생각하면), ruby와 같으 스크립트 언어 사용

stdio

  • history

    • C 언어로부터 유래
    • 추상화된 입출려 계층이 필요. 장치에 대해 투명성(transparency)을 제공
    • block device나 character device는 모두 abstract stream(byte base)로 설계 가능
    • 유닉스 계열의 /dev/*를 오픈할 때 fd를 부여

  • stdio

    • stdin standard input 0
    • stdout standard out 1
    • stderr standard error 2
    • 추상화된 장치이므로 논리적으로 연결, 해제 등이 가능
    • 기본적으로 연결된 stdin은 특정 파일에 연결시킬 수 있음

  • redirection

    • /> 표준 출력을 파일로
    • />/> 표준 출력으 기존의 파일에 덧붙이거나 없으며 새 파일
    • /< 표준 입력을 파일에서 입력
    • /</< 표준입력에 텍스트르 입력

<&# | 표준 입력을 파일에서
#> | fd #번을 파일로
&> | stdout, stderr을 파일로
>&# | 출력을 fd #으로
a | b | 리다이렉션파이프, a의 표준출력은 b의 표준입려으로 연결됨

  • pipe redirection

>(cmd) | anonymous pipe로 만들어 cmd 명령에 보내어 출력 cmd 실행 결과는 임시 anonymous pipe인 /dev/fd/60 으로 매핑된다.
<(cmd) | cmd 명령 결과를 anonymous pipe로 보내어 읽어들인다. cmd 실행 결과는 임시 anonymous pipe인 /dev/fd/63 으로 매핑된다.

괄호 기능 총정리

  • 소괄호 parentheses

$(string) | string 문자열을 명령어로 인식하여 sub-shell로 실행. 서브셸에서 선언 or 변경된 값은 서브셸 스코프에서만 유효. string 앞뒤 공백 허용
$((string)) | string 해당하는 문자열을 수치 연산을 인식. expr, let으로 대치 가능.(expr은 외부 명령으로 지양) string 앞뒤 공백 허용

  • 중괄호 braces

{… , … , …} | brace expansion. a{1,2,3}b = a1b a2b a3b
{ …; …; } | 현대 셸에서 실행함. 공백, 마지막 세미콜론 생략시 에러 발생

  • 대괄호 brackets
    […] | 배열 변수의 인덱스
    [ expression ] | 테스트 구문(expression 부분의 true, false를 테스트)

  • sub-shell 에서 PID, PPID 를 얻어야 하는 경우 변수가 치환되는 시점이 스크립특 해석되느 시점에 결정되기 때문에 오동작이 될 수 있다.

    • 변수 할당

${varname:=default} | varname이 null이면 default를 할달. 아니면 기존 값으로 치환
${varname:-default} | varname이 null이면 default를 반환. 할당 안함
${varname:+alternat} | varname이 nullㅇ 아니면 alternat르 반환. null이며 그대로. 할당 안함
${varname:?error} | varname이 null이면 에러 발생 후 error 메시지의 값을 화면에 출력. return code는 1

${varname#pattern} | 전방 탐색으 pattern과 매칭되는 varname의 값이 삭제됨
