VIM Editor

vim editor에 대하여

  • 포스트를 마크다운 문법과 본문의 쉘 문법이 섞여 정상적으로 나타나지 않을 수 있음을 주의바랍니다.

vim 시작

  • vim [filename] 으로 실행
  • find . -name “*.txt” | vim -
  • vi의 작동모드는 일반모드(normal mode), 입력모드(insert mode), 명령행모드(command-line mode), 비주얼모드(visual mode)가 있음
    • normal mode에서 a,i,o 입력 -> insert mode 반대는 ESC 입력
    • normal mode에서 : 입력 -> command mode 반대는 ESC, 실행
    • 노말모드에서 단축키로 실행
    • a: append는 현재 커서 위치에서 한칸 뒤로 이동한 후 입력 모드로 전환 A는 행의 가장 마지막
    • i: insert는 현재 커서 위치에서 입력 모드로 전환 I는 행의 맨 앞
    • o: 커서 위치 아래에 행이 생기면서 입력모드로 전환
    • !: force 명령으로 강제 시행을 의미
  • 커서 이동
    • 일반모드에서 화살표키 대신에 h(left), j(down), k(up), l(right) 을 이용하여 커서를 이동
    • ^: 행의 맨 앞으로 이동(caret)
    • $: 행의 맨 끝으로 이동
    • <CTRL-B>: page up backward
    • <CTRL-F>: page down forward
    • <CTRL-U>: 1/2 page up upward
    • <CTRL-D>: 1/2 page down downward
    • :# -> #행으로 이동
    • #gg: #행으로 이동 # 생략 시 맨위로 이동
    • #G: #행으로 이동 # 생략 시 마지막으로 이동

buffer 사용

  • vi에서 삭제 = 임시 버퍼에 잘라내기
    • x: 커서에 위치한 문자 삭제 = delete키
    • dd(:d): 현재 행을 삭제 (3dd는 세줄 삭제)
    • D: 현재 컬럼 위치에서 현재행의 끝부분까지 삭제
    • J: 아래 행을 현재 행의 끝에 붙임(아래 행의 앞부분 공백은 제거됨)
  • paste
    • p: 현재 행에 붙여 넣습니다.(put)(5p 는 5번 반복 붙여넣기)
    • pu: 개행 문자가 포함된 경우에는 현재 행의 아래에 붙여 넣습니다.
    • P: 현재 행의 위쪽에 붙입니다.(대문자)
  • copy
    • yy, :y, Y: 현재 행을 레지스터에 복사(yank)
  • u: (undo) 바로 이전에 행한 명령 한 개를 취소
  • <CTRL-R>: (redo) 바로 이전에 취소했던 명령을 다시 실행
  • .: 이전 명령어 반복

range

  • 명령행 모드에서 범위를 지정해서 명령
  • :20d - 20번 행을 삭제
  • :10,25d - 10~25번 행을 삭제
  • :10,$d - 10~마지막 행까지 삭제
  • :&y - 문서 전체를 복사. %는 1,$와 동일
  • :.,+20y - 현재 행부터 아래로 스무행을 복사
  • :-10,+5d - 현재 행부터 위로 10행, 아래로 5행, 총 열여섯 행을 삭제
  • :40pu - 40번 행에 레지스터의 내용 붙여넣기
  • 범위 연산 meta character
    • .: 현재 행을 의미(relative path의 현재 디렉터리와 유사)
    • $: 마지막 행을 의미 (REGEX의 $와 유사)
    • +#: 현재 위치에서 #만큼 아래 행을 의미
    • -#: 현재 위치에서 #만큼 위 행을 의미
    • %: 문서(파일) 전체를 의미

visual mode

  • mouse의 drag 기능을 대신
  • v: (소문자) 일반 비주얼 모드로 현재 커서 위치에서 블록을 지정
  • V: (대문자) visual line mode로, 현재 커서가 위치한 행에서 행 단위로 블록을 지정
  • <CTRL-V>: visual block mode, 열(column)단위로 블록을 지정(ctrl + q 로 사용하기도 함)
  • column editing
    • 특정 열에 문자열을 삽입/교체하는 경우
    • <CTRL-V>, 에디팅, ESC*2 하면 완료
    • I: insert, A: append
    • c: change
    • ~: switch case
  • gv: previous highlighted text 영역 불러오기
  • o: hightlighted text 블록의 시작, 끝 이동

clipboard

  • vi: vi 호환의 vim 이며 vim 종류 중에 가장 적은 기능을 가지고 있다. compact vim, tiny vim 이라 부름
  • vim: 일반적인 vim을 의미. vi에 추가적인 기능과 플러그인 사용이 가능. enhanced vim이라 부름. 프로그래밍이 많이 사용됨.
  • vimx: 가장 많은 기능을 가진 vim. vim + clipboard, X window clipboard 기능 포함.
  • $ vim --version | grep clipboard 명령어로 clipboard 기능이 포함되어있는지 알 수 있다.
  • 기능이 포함되어 있는 경우 아래 내용을 ~/. vimrc 파일에 추가한다. X window에서 개발 할 경우 이 기능은 유용하다.

vim options

  • ~/.vimrc 파일에 옵션 저장 가능
    • :set - 현재 옵션 설정을 보여줌
    • :set all - 모든 옵션 설정을 보여줌(default 옵션까지 출력)
    • :set [no]name - name에 해당하는 옵션을 on/off 함
    • :set name! name옵션의 on.off를 toggle 함
    • :set name=value name옵션에 value의 값을 할당
  • 주로 사용하는 옵션
    • nu: (number) 화면에 행번호 표시. nu는 초보로 의심 받을 수 있음??
    • rnu: (relativenu) 현재행을 기준으로 상하의 상대적 행번호 표시.
    • ai: (autoindent) 자동 들여쓰기
    • cindent: C언어 스타일의 들여쓰기
    • ts=value: (tabstop) 화면에 표시 될 탭 크기를 value로 지정
    • sw=value: (shiftwidth) 자동 블록 이동시 열의 너비
    • fencs=value: (fileencodings) 지원할 파일 인코딩 방식 리스트 e.g. set fencs=ucs-bom,utf-8,korea
  • colorscheme
    • :coloscheme + ‘tab’ 으로 선택 가능
    • jellybean 과 같은 커스텀 색도 별도 추가할 수 있음.

vim manual

  • :help word + <CTRL-D> word가 포함된 명령어를 볼 수 있음
  • :help 뒤에 여러가지 모드를 추가해 설명을 확인 할 수 있음.
    • 없음 - :help x 일반 모드
    • i_ - :help i_CTRL-N 입력모드
    • : - :help :w 명령행모드
    • v_ - :help v_u 비주얼모드
    • - :help -r vim 실행인수
    • ' - :help ‘tabstop’ 옵션
    • c_ - :help c_CTRL-B 명령행 모드 특수키
  • help 모드에서 명령어
    • ^] <CTRL-]>: jump to the definition of the keyword under the cusor
    • ^T <CTRL-T>: jump to older entry in the tag stack
    • :tags: show the contents of the tag stack. The active entry is marked with a ‘>’
    • ^W^W (<CTRL-W>*2): move cursor to window below/right of the current one

vim error

  • 파일을 중복해서 오픈 한 경우: 오류 메세지 process ID에 still running이라고 써있을 경우 Q(quit), O(read only) 를 선택
  • vim crash로 swapfile이 제거되지 못한 경우: 오류 메세지 process ID에 pid만 써있을 경우 R(recovery)을 선택

문자열 관련 기능

  • center, right, left
  • 문자 1개를 검색
    • fc: 문자 c를 전방 검색
    • Fc: 문자 c를 후방 검색
    • ;: 최근 검색을 재검색
    • ,: 최근 검색을 반대방향으로 재검색
  • 단어 검색
    • /string: string을 전방탐색
    • ?string: string을 후방탐색
    • *: 현재 커서에 위치한 단어를 전방탐색함
    • #: 현재 커서에 위치한 단어를 후방탐색함
    • n: 다음 탐색 결과를 찾음(전방탐색의 경우 전방으로, 후방탐색의 경우 후방으로 이동)
    • N: n과 반대방향으로 다음 탐색
    • %: 괄호의 짝을 찾아줌
    • case-sensitive searching
      • \c 접두어를 추가할 경우 e.g. /\cinter 로 실행하면 inter의 대소문자 무시 \C(대문자c)는 반대 기능
      • :set ignorecase 는 ignorecase 활성화 반대는 :set noignorecase
    • /[a-g]re: are, bre,cre ~ gre 를 모두 찾음. REGEX 문법을 따름.
    • :nohls - :set nohls 검색 후 밝게 표시된 단어 옵션을 해제
    • \M magic이라 불린다. 해석을 금지하는 접두어. \M[section] 이렇게 하면 REGEX로 해석을 안하고 그냥 문자열로 인식
  • 문자열 교체
    • sed의 기능이 import 된 것. sed 문법과 동일
    • :[range]s/<찾는 문자열>/<교체할 문자열>/<옵션> separator로 slash가 주로 사용되지만 다른 문자를 사용해도 무방(예: 콤마)
    • 명령어 옵션
      • g: (global) 검색된 문자열 모두를 교체. 이 옵션 생략시 처음 1개만 교체
      • i: (ignore case) 대소문자 무시
      • c: (confirm) 교체 할 때마다 확인
      • e: 교체 과정 중 에러 무시
    • 찾으려는 문자열에 /가 들어있을 경우 escape문자를 추가한다.
      • :%s//home/linux//home/unix/g 를 아래 처럼 써도 무방하다.
      • :%s,/home/linux,/home/unix,g
    • 특수 문자의 교체
      • 개행문자 DOS(CR+LF) UNIX(LF)
      • CR 입력: ^V^M <CTRL-V> 대신 <CTRL-Q> 사용 가능
      • :%s/CR입력//g 혹은 :%s/\r//g
      • 실전에서 :set ff=dos 혹은 :set ff=unix 로 설정 후 저장하는 방법 씀
    • ASCII를 코드값으로 입력
      • <CTRL-V> ###: ###에 10진수를 사용하여 ASCII 입력
      • <CTRL-V> o###: ###에 8진수를 사용하여 ASCII 입력
      • <CTRL-V> x###: ###에 16진수를 사용하여 ASCII 입력
      • man ascii 로 아스키코드 확인 가능
    • 숫자 증감: 숫자 위에서 <CTRL-A> 누르면 증가 <CTRL-X> 누르면 감소

파일 관련 기능

  • buffer: 파일을 편집하기 위한 임시 공간. 파일명 없이 vim을 실행하면 이름이 없는 buffer에 작업. :w myfile.txt 처럼 저장하면 buffer에 있는 내용이 저장이 됨
  • register: 텍스트 일부를 저장하고 있는 임시 공간. dd 사용하면…
  • vim에서 다른 파일을 편집하려고 할 때
    • :e [filename] 파일을 편집모드로 오픈한다. 파일이름을 생략하면 현재 파일을 다시 오픈한다.
    • :e #[count] count번째 파일을 오픈한다. count가 생략되면 바로 인전 파일이다.
    • :find filename 파일을 검색하여 오픈. 매칭결과가 여러개이면 에러메시지를 출력
    • ctrl + ^ 명령어 단축키 ":e #"과 동일
  • 한번에 여러개 열 수 있음
    • $vim file1 file2 file3
    • :n입력하면 다음 파일 :N 입력하면 이전 파일
  • 종료 기능
    • q[!] 현재 창 종료. !는 강제종료
    • :qa[!] 모든 창 종료. 복수개의 창을 열었을 때 유용
    • :wq 저장하면서 종료 :x 와 동일 요즘은 ZZ 사용
    • :wqa 저장하면서 종료를 모든 창에 대해서
  • 파일 저장
    • :w [filename] 파일이 지정되면 해당 파일에 쓰기를 지정(복사본) 사본을 만든 경우 편집중인 파일은 원래 파일
    • sav file 현재 파일을 다른이름으로 저장. 편집중인 파일도 새로 저장된 파일로 교체
    • :up 변경된 점이 있는 경우만 :w 명령어 실행
    • :x up+quit 일반모드에서는 ZZ
    • :10,50w history 10~50 번 행을 history라는 파일로 저장
    • :.,+10w history 현재행에서 아래로 10번행을 history라는 파일로 저장
    • :10,$w history 10~끝 까지 저장
    • :%w history 전체를 history 파일로 저장, 즉 %는 있으나 없으나 결과가 같음
  • netrw: vim으로 디렉터리 열기(browsing)
    • :e . 하면 현재 디렉토리 열기
    • <Enter> 파일을 현재 창에 열기
    • i 파일 표시 방법 변경(한줄, 파일정보도 함께, 와이드 형식, 트리방식)
    • s 정렬 방식을 바꿈(이름순, 시간순, 크기순)
    • o 커서 위치 파일을 수평 분할된 새 창으로 열기
    • v 커서 위치 파일을 수직 분할된 새 창으로 열기
    • p 커서 위치 파일을 미리보기 창으로 열기. 창닫기는 ctrl + w z나 :pclose
    • t 새로운 탭으로 분할하여 열기
    • - 상위 디렉터리로 이동
  • 복수의 파일 다루기
    • split horizontal CTRL-W s / :sp [file]
    • split virtical CTRL-W v / :vs [file]
    • 창 이동
      • CTRL-W CTRL-방향키
      • CTRL-W CTRL-W 현재 창에서 오른쪽 방향으로 이동
      • CTRL-W CTRL-P 바로 이전에 이동한 창으로 이동
    • 창 크기
      • CTRL-W = 모든 창의 크기를 동일하게 조절
      • CTRL-W [#]+ #크기만큼 크기를 키움. 숫자 생략시 1 키움
      • CTRL-W [#]- #크기만큼 크기를 줄임. 숫자 생략시 1 줄임
    • split diff
      • vim -d file1 file2
      • do, dp 로 바로 변경 사항 가져오기
    • tabpage
      • vim -p file1 file2 file3
      • :[#]tabe[dit] file #번재 탭에 파일을 연다. #생략시 현재 탭 뒤에 생성. 번호는 0번부터
      • :[#]tabnew file #번째 위치에 비어있는 탭 생성
      • :[#]tabc[los] #번째 탭을 닫음. #생략시 현재 탭 닫음
      • :[#]tabn[ext] [#]gt [#]<CTRL-PageDown> 다음 탭으로 이동, 일반모드의 gt와 동일
      • :[#]tabp[revious] [#]gT [#]<CTRL-PageUp> 이전 탭으로 이동, 일반모드의 gT와 동일
      • :tabm[ove] [#] #번째 탭으로 이동(0부터 시작) #생략시 가장 오른쪽으로 이동
  • buffers
    • 아직 저장 전이면 이름없는 공간으로 버퍼라 부른다
    • :file :buffer 명령어로 버퍼를 볼 수 있다.
    • % 현재 편집 중인 버퍼
    • # 바로 이전에 열었던(혹은 열 예정인) 버퍼
    • a 활성화된 버퍼. 창 분할 기능을 쓸 경우 여러 파일에 나타남
    • + 변경된 부분이 있는 버퍼
  • 커서 아래 파일명을 인식하여 오픈하는 기능
    • 파일 명에 커서를 가져다 놓고 gf(이동),
    • C 언어 소스코드에 include 파일에 커서 놓고 gf 입력
    • <CTRL-W> f 커서 위치 파일명을 분할된 창에 열어줌
    • <CTRL-W> gf 커서 위치 파일명을 탭에 열어줌
    • <CTRL-^(6)> 이전 파일로 되돌아가기

파일 인코딩

  • fileencodings (fencs)
    • 파일을 읽을 때 확인할 encoding list를 설정. .vimrc 에 list로 설정. 순서 중요
    • set fencs=value e.g. fencs=usc-bom,utf-8,korea,latin-1
    • BOM(Byte Order Mark) UCS(Universal coded Charater Set) 주로 생략하나, 관습적으로 옵션 설정시 앞부분에 넣어줌
    • 사용 가능 문자 셋 :help encoding-values
      • utf-8, utf8, usc-bom, korea(이건 별칭으로 유닉스에서는 euc-kr로 윈도에서는 cp949로 바뀜), euc-kr, cp949, japan, latin1, ansi(영문 아스키 방식)
    • :set fenc=korea 로 하면 현재 사용 문자 인코딩을 설정하는 것
    • linux to window 파일을 가져갈 때 UTF-8 을 지원하는 에디터가 아닐 경우 변환을 거쳐야함.
      • :set ff=dos :set fenc=korea :wq 명령어를 통해 변환 :sav newfilename.txt 로 다른이름 저장
    • window to linux 파일을 가져올 때 fencs가 잘 설정되어있으면 자동 변환됨. 이후 저장은 dos 형태로 되기 때문에
      • :set ff=unix :set fenc=utf8 :wq 명령어를 통해 리눅스에서 추후 사용시 편하도록 변환 :sav 로 다른 이름 저장

기타 편리한 기능

  • 단어 경계 이동

    • w, e 키를 이용해 이동 w는 단어의 앞에 멈춤. e는 단어 뒤에 멈춤
    • 0 0번째 열 이동
    • ^ 공백이 아닌 실제 내용이 있는 시작 열
    • $ 마지막 열(행의 끝)
    • b w와 비슷하나 진행 방향이 반대
    • W, E, B 해봐라

  • 괄호, 문단, 블록 단위 이동

    • % 가장 가까운 괄호 짝으로 이동
    • (, ) 문장 단위의 시작 위치, 끝 위치로 이동
    • {, } 문단 단위의 시작 위치, 끝 위치로 이동
    • [[,]] 블록 단위의 시작 위치, 끝 위치로 이동

  • abbreviation

    • 특정 단어 입력시 대체 입력하는 기능
    • "내멜"을 입력하면 실제 이메일이 입력되는 기능
    • ia는 insert mode 에서만 작동. ca는 command mode 에서만 작동
    • ab 내맬 abcdvvv@gmail.com
    • ab projectsrc /export/2/local/pjr2/mds/src
    • ia 시간0 <C-R>=strftime(“%Y.%m.%d-%H:%M:%S”)<CR>
    • ia tlrks1 <C-R>=strftime(“%c”)<CR>
    • ca 기능을 이용하면 한글상태 오타를 변환할 수 있다.
      • ca ㅈ w
      • ca ㅈㅂ wq
      • ca ㅌ x
      • .vimrc 파일이 EUC-KR로 되어있으면 UTF-8 무서 편집시 작동하지 않을 수 있다. fenc=euc-kr은 안됨!
        명령 설명
        :ab [lhs] 현재 설정된 모든 약어 목록을 출력. lhs에 약어 지정시 해당 약어 정보만 출력
        :ab {lhs} {rhs} 약어 lhs를 rhs 로 변환
        :unab {lhs} 약어 lhs 해제
        :abclear 설정된 모든 약어 해제
        :ia {lhs} {rhs} ab와 기능은 같지만 입력 모드에서만 작동
        :ca {lhs} {rhs} ab와 기능은 같지만 명령행 모드에서만 작동

  • key map

    • nmap <단축키> <명령> normal mode 에서만 작동
      • nmap <F2> :up<CR>
    • imap <단축키> <명령> insert mode 에서만 작동
      • imap <F4> abcdvvv@gmail.com
    • map <단축키> <명령>
    • vmap, cmap 도 가능
    • .vimrc에 nmap <F3> :tabnext<Enter> nmap <F4> gT 이렇게 적을 놓으면 됨
    • autocmd 특정 상황에서 자동으로 실행할 명령
      • 예시 *.c 파일을 열 때 자동으로 실행할 명령
