Linux Package System

  • Redhat Package System(RPM)
    • rpm, yum, dnf
  • Debian Package System(DEB)
    • dpkg, apt-get, aptitude, apt
  • Package
    • 시스템을 구성하는 파일의 묶음
    • 패키지 관리자에 의해서 읽혀지는 pre-built 파일들
    • 관리의 편리함을 제공한다.
  • dpkg file 구조
    • strace_4.5.20_2.3_amd64.deb
    • {package name}_{version & release}_{arch}.deb
  • dpkg
    • dpkg, dselect가 초기에 사용되었으나 dependency 해결과 검색이 어렵고 네트워크 설치 지원 못하여 apt가 등장
    • dpkg -l {package name} 패키지 리스트 확인
    • dpkg -s {package name} 패키지 상태 확인
    • dpkg -S {file} 파일명으로 패키지 검색
    • dpkg --audit 오류 확인

apt

  • advanced package tools
    • dpkg를 래핑한 front-end tool
    • 네트워크 설치 지원(mirror 탐색 가능)
    • dependency 탐색 및 설치 가능
  • source list
    • apt가 package file 을 다운 받는 곳의 정보
    • /etc/apt/sources.list 직접 편집 권장 하지 않음
    • /etc/apt/sources.list.d 에 *.list 파일명으로 추가 권장
    • apt edit-sources 명령어로 편집 가능
    • deb [option1=value1 option2=value2] ury suite [component1] [component] […]
    • deb-src [option1=value1 option2=value2] ury suite [component1] [component] […]
    • uri: deb 패키지를 제공하는 사이트의 URI
    • suite: distribution codename 디렉터리 이름 (==dist. version)
    • component: suite의 구성 요소 및 라이선스 종류별 분류
      • main(officially supported, free), restricted(restriced free license), universe(community-maintained, not official) etc.
    • sudo select-editor 명령을 이용해 시스템 에디터를 설정
    • sudo apt edit-sources kakao.list 명령어 입력 후 아래 내용 입력 후 sudo apt update 입력하여 목록 갱신
  • apt list [options] [package pattern]
    • 해당 버전에서 제공되는 패키지의 최신 목록 출력
    • apt list --installed 설치된 패키지 리스트
    • apt list --upgradable 업그레이드가 가능한 패키지 리스트
    • apt list --all-versions 모든 버전의 패키지 리스트
    • apt list bash* bash로 시작하는 패키지 리스트
  • apt search [-n] <regex>
    • 패키지를 키워드로 검색. 키워드는 REGEX 로 입력
    • apt search bash description에 bash가 들어간 경우까지 검색
    • apt search -n bash 패키지 이름 중간에 bash가 있어도 검색
    • apt search -n ‘^bash’ 패키지 이름 시작 부분에 bash가 있는 경우 검색
    • apt-cache pkgnames name 패키지 이름으로만 검색
  • apt show <package name>[=version]
    • 패키지 정보 출력
    • apt show bash bash 정보 출력
    • apt list --all-versions bash 모든 버전 정보 출력
    • apt show bash=4.4.17-2ubuntu1 특정 버전 정보 출력
  • apt <remove|purge|autoremove> <package>[=version]
    • apt remove 패키지만 삭제 (config file은 남겨둠)
    • apt purge 패키지 삭제(config도 삭제)
    • apt autoremove 의존성이 깨지거나 버전 관리로 인해 쓰이지 않는 패키지를 자동 제거
  • apt install 중 에러가 날 때
    • 일반적으로 apt sources.list 설정에서 components에 universe를 빼먹는 경우 생긴다. 가장 좋은 방법은 universe를 추가하는 방법이지만 downgrade도 하는 방법을 알아두면 좋을거라
    • apt list --all-versions 패키지 로 검색
    • apt install curl libvurl3-gnutls=7.47.0.1ubuntu2 로 해당 버전을 설치하여 다운그레이드
  • aptitude
    • TUI 방식의 apt
    • ncurses based tool
    • 잘 안쓴다…

Linux Network System

  • 네트워크 기본 용어
    • hostname: primary hostname, FQDN
      • abc.redhat.com 에서 given name + family name에 비유하면 redhat.com은 family name으로 domain address라 불리기도한다.
      • domain을 제외한 호스트 이름 혹은 domain을 포함한 FQDN
      • FQDN(Fully Qualified Domain Name) 도메인 내에서 유일하게 구별 가능한 이름. DNS server에서 정함
      • 도메인 주소는 체계적인 구조를 가짐 devel.fclinux.or.kr
      • localhost는 항상 자기 자신을 의미하는 주소와 매핑되는 특별한 hostname. (IPv4= 127.0.0.1, IPv6= ::1)
    • TCP/IP: IP address(ipv4, ipv6), subnet mask, gateway
      • IPv4 32bit 주소 체계. 8bit씩 끊어서 xxx.ooo.xxx.ooo
      • IPv6 128bit 주소 체계. 무지 많다.
      • IPv4-mapped IPv6 ::ffff:IPv4_addr
      • dual stack IPv4와 IPv6를 혼용해서 사용하기 위한 설계. 대부분 OS 지원 (win xp 제외)
      • A,B,C 클래스 뭐시기는 없어짐
      • CIDR(Classless Inter-Domain Routing) IP클래스와 상관럾이 서브넷을 지정하여 자르는 것을 의미. 서브넷 마스크 비트 크기.
      • public IP 공인주소(인터넷에서 유일한 주소), private IP(인터넷에 직접 연결되지 않은 주소)
    • NIC: Network Interface Card == 랜카드
    • Wired Network (Wired connection): 유선 네트워크(유선 연결)
    • Wireless Network (Wireless connection): 무선 네트워크(무선 연결)
    • LAN: Local Area Network
    • WAN: Wide Area Network
    • SELinux(Security Enhanced Linux) 커널 레벨에서의 중앙 집중식 보안 기능
    • SELinux 보안 레벨
      • enforcing(강제): 보안설정에 걸리면 강제로 막음(많은 보안 지식필요)
      • permissive(허가): 보안설정에 걸리면 허용하되 로그 출력(실습시 사용 권장)
      • disable(비활성): SELinux 사용하지 않음

Linux Network Configuration Legacy

  • Legacy방식과 NetworkManager 방식 있음
    • debian legacy config: /etc/network/interfaces
    • RedHat lagacy config: /etc/sysconfig/network-scripts/ifcfg-*
    • 이제는 NetworkManager로 통일 됨
  • Debian Legacy
    • 쓰지 마라. 혹은 legacy를 사용하는 커스텀 리눅스를 쓸 경우
    • /etc/network/interfaces 파일에 설정함
    • 이런저런 기능들이 있으나 안쓰인다. 이젠 NetworkManager 쓴다.
    • /etc/init.d/networking restart 이런 명령어 쓰지마라
    • ifup, ifdown 이런거 쓰지마라
    • /etc/default/networking 설정 파일 쓰지마라

Linux NetworkManager

  • NetworkManager는 deamon으로 작동하면서 network configureation을 수행하고, 자동으로 network connection을 관리(연결 감지, 해제, 재시도)를 수행한다. Dbus 기반으로 동적 상태를 감지할 수 있기 때문에 다른 애플리케이션이나 deamon들에게 네트워크 정보를 제공하거나, 관리를 위한 권한을 줄 수도 있다. 또한 명령어를 사용하여 systemd 기반의 다른 Linux distribution들에게도 동일한 방식의 경험을 제공할 수 있다. Ethernet, Wi-Fi, Mobile broadband 등 다양한 기능들에게 플랫폼을 제공하므로, 네트워크 연결 관리가 좀 더 쉬워진다.
  • nmcli (Network Manager CLI tool) 네트워크 관련 대부분의 기능을 가지고 있다.
    • nmcli g[eneral] 현재 상태 알려줌
    • nmcli n[etworking] 상태 조회
    • nmcli n on/off 네크워크 켜기/끄기
    • nmcli d[ev[ice]] 장치 조회
  • networks device naming
    • eth#[:n] eth0, eth1, eth0:0 등 으로 나타남. 옛날 표현 방식
    • Consistent Network Device Naming(2009)
      • prefix en(ethernet), wl(wireless lan), ww(wireless wan)
      • o<index> on-board device index number
      • s<slot>[f<function>][d<dev_id] hotplug slot index number
      • x<mac> MAC address
      • p<bus>s<slot>[f<function>][d<dev_id] PCI geographical location
      • p<bus>s<slot>[f<function>][d<dev_id][…][c<config>][i<interface>] USB port number chain
      • enp5s0 -> ethernet & pci number 0
  • nmcli < g| n | r | c| d > general, networking, radio, connection, device
  • nmcli c[connection] s[how]
    • nmcli c s ens33 소문자로 나오는건 설정된 값(offline 일때도 보임). 대문자는 할달된 값(online 일때만 보임)
    • nmcli con down “connection name” connection 끊기 후 nmcli cs “connection name” 하면 IP4.* 이부분이 안나옴.
    • nmcli con up “connection name” connection 연결 후 nmcli cs “connection name” 하면 IP4.* 이부분이 나옴.
  • nmcli 속성 변경
    • nmcli c modify “connection name” connection.id ens33 -> connection id를 device name으로 변경
    • nmcli c mod “c name” ipv4.method manual ipv4.address 192.168.xxx.xx/24 ipv4.gateway 192.168.xxx.2 +ipv4.dns 8.8.8.8 ipv4.dns를 추가 아이피와 게이트웨이 변경
    • nmcli c down “name” && nmcli c up “name” 재시작을 한 후 nmcli c s “name” 으로 상태 확인
    • nmcli c mod “name” +ipv4.address 192.168.xxx.xx/24 virtual ip 추가
    • nmcli c mod “name” -ipv4.address 192.168.xxx.xx/24 ip 삭제
    • nmcli c del “name” 기존 설정 삭제 후 nmcli d s 확인 해 본다
    • nmcli c add con-name “connection name” ifname “interface(device) name” type ethernet ip4 192.168.xxx.xx/24 디바이스 추가
    • nmcli c mod “name” +ipv4.dns 8.8.8.8 DNS 추가
    • nmcli c s “name” | egrep’(ipv4.addr|ipv4.gateway|ipv4.dns)’
    • nmcli dev connect “device name” 디바이스 자체가 연결끊어짐 상태인 경우 액티브 상태로 설정
    • connect 가 실패할 때 nmcli networking 으로 확인해보고 nmcli networking on 으로 켜줌

  • address filter
    • > gt(greater than), < lt(less than), >= ge(greater than or equal), <= le(less than or equal), == eq(equal), != ne(not equal), and(and operator), or(or operator)

    • ss -n ‘( dport = :ssh or sport = :ssh ) and src 192.168.xxx.xx’ parenthesis 공백에 주의
    • ss -nt ‘dst :22 or dst :80’
    • ss -n ‘dport > :8000’, ss -n ‘dport gt :8000’
    • ss -n ‘dport > :8000 and dport < :20000’

Querying

  • ping

    • 상대 호스트 응답확인(호스트 존재 여부 확인)
    • ping [-c count] [-i interval] [-s size] [-t ttl] target
    • 마지막에 간략한 통계정보를 확인 할 수 있다. 표준편차 값이 너무 크면 문제가 있을 경우가 있음.

  • traceroute

    • 패킷 도달 경로를 확인
    • traceroute -n 63.99.207.68

  • arp

    • ARP(address resolution protocol) 테이블 조회 명령어
    • IP(L3) 와 MAC(L2) 주소의 매칭 테이블
    • arp -s <MAC address> <IP address> static arp address를 사용하는 명령어
    • NIC 교체후 통신이 실패하는 경우? 보안을 위해 고정 ARP 테이블을 사용하는 경우에 발생. ARP table을 확인하여 NIC 정보 일치 여부 확인

  • resolver : name service

    • IP address 혹은 hostname(FQDN, alias)를 해석
    • nameserver를 의미
    • /etc/resolv.conf 에 지정됨 /etc/nsswitch.conf에 해석기 및 우선 순위 지정. 수동 편집 지양.
    • resolver client
      • nslookup -> deprecated
      • dig 최신 명령어 e.g. dig www.naver.com, dig @8.8.8.8 www.naver.com
    • nameserver 유명 네임서버는 몇개 알아두자
      • 1.1.1.1 Cloudflare DNS
      • 208.67.222.222, 208.67.200.200 CISCO openDNS
      • 8.8.8.8 google DNS

  • ethtool: query or control network driver and hardware settings

    • 간혹 시스템이 느려지는 경우는 duplex 문제일 가능성이 크다
    • ethtool -s “name” speed 1000 duplex full 이렇게 수동 설정해줘. 다른 프로그램이 설정을 바꾸는 경우 발생할 수 있음.
    • ethtool -s “name” speed duplex full autoneg off auto-negociation 설정 바꾸기
    • ethtool -S “name” 장비 관련 통계 정보 조회
    • ethtool -k <dev> show offloading CPU가 할일을 덜어주는 것
    • ethtool -K <dev> <param> <on|off>

Network Tools

  • ssh

    • secure shell 통신 구간 암호화
    • telnet 서비스는 안씀
    • Linux는 openssh 사용
    • sshd는 ssh deamon으로 ssh server를 의미
    • ssh는 ssh client이며 cli 명령어로 사용됨
    • windows에서는 putty, mobaXtrem 등 사용

  • sshd 서버 준비

    • 서버 설치 여부 확인 apt list openssh*, rpm -qa openssh-server
    • 서비스가 실행중인지 확인 systemctl status sshd
    • 22번 포트 확인 ss -nlt, ss -nltp
      • * 주소는 INADDR_ANY 0.0.0.0을 의미 모든 IP & :::22
    • 방화벽에 22번 포트가 허용되어 있는지 확인 - iptables -nL
      • ufw status/enable/disable/-h 명령어로 방화벽 확인 ufw allow 22/tcp, ufw allow http/tcp
    • sshd 서비스 정지시 - systemctl start sshd
    • 부팅 시 자동실행 - systemctl enable sshd [–now]

  • ssh client

    • ssh [-p port] [username@]<host address> UNIX like 운영체제에서 사용 명령어
    • ssh 192.168.xxx.xx
    • ssh linuxer@192.168.xxx.xx
    • ssh -p 20022 192.168.xxx.xx
    • windows 계열은 putty, mobaXtrem 등을 사용

  • ssh-keygen

    • ssh-keygen -N “”
    • -N new passphrase 개인용에서는 암호를 쓰지 않지만, 기업에서는 꼭 “” 대신에 패스워드 사용
    • key 위치는 ~/.ssh
    • ssh-copy-id linuxer@192.168.xxx.xx 생성한 키를 상대편(remote)에 ~/.ssh/authorized_keys에 복사
    • ssh -t linuxer@192.168.xxx.xx w w라는 명령을 remote에서 실행하고 결과를 가져오는 방법

  • curl

    • URL을 기반으로 통신하는 기능 제공
    • libcurl 라이브러리 제공
    • curl [options] <URL>
    • curl -O URL 파일로 저장
    • curl -o nate.html URL 지정한 파일명으로 저장
    • curl -C - -O URL 파일 다운로드 -C는 continue 옵션
    • curl v2.wttr.in/Seoul
    • curl https://api.exchangeratesapi.io/latest?base=USD

  • wget URL curl과 대부분 비슷하나 curl 이 더많은 기능가짐 파일 다운로드에 특화된 기능 존재 web site miirroring 기능 있음.

  • nc

    • netcat
    • network 기능이 가능한 cat
    • server, client의 양쪽 기능이 가능. 간이서버 클라이언트로 사용. 바이너리 통신 가능.
    • nc -k -l 5000
    • echo Hello | nc 127.0.0.1 5000
    • nc 127.0.0.1 5000

    • 아니면 bios에서 막힌 경우가 있음. 랩탑에서 와이파이 끄는 기능 혹은 sleep 기능이 hardware block을 하기때문에 이런 부분 확인 필요.
    • nmcli dev [list] 에서 type wifi 부분 확인 가능
    • nmcli dev wifi 명령으로 접속 가능한 장치 리스트 확인 가능. nmcli dev wifi rescan 으로 다시 장치 스캔.
    • nmcli dev wifi connect “ssid” password “pass” 명령어 입력 후. nmcli d 로 연결 확인
    • nmcli d disconnect “device name”

  • WiFi hotspot

    • nmcli c add type wifi ifname ‘*’ con-name “hotspotname” autoconnect no ssid “ssid” 핫스팟 생성!
    • nmcli c mod “hotspotname” 802-11-wireless.mode ap 802-11-wireless.band bg ipv4.method shared ipv4.address 192.168.100.1/24(hotspot ip)
    • nmcli c mod “hospotname” 802-11-wireless-security.key-mgmt wpa-psk 802-11-wireless-security.psk “password”
    • nmcli c up “hostspotname” 핫스팟 시작
    • nmcli c s 로 핫스팟 확인 가능
    • nmcli -p c s “hotspotname” 핫스팟 정보 확인 가능
    • nmcli c down “hotspotname” 핫스팟 종료

  • hostapd

    • ap로 구동을 위한 기능 제공 deamon
    • NetworkManager 보다 더 많은 기능을 사용하려면 hostapd를 사용
    • 5GHx AP 구동시 regulartory domain rule에 의해 region 설정시 일부 채널이 블록 되거나 passive로 작동 될 수 있다.
    • 이를 해결하기 위해 kernel module 수정이 필요하다.

File System

  • FS는 OS의 매우 큰 부분을 차지한다. read, wirte 작업을 수행

  • FS는 넓게 보면 데이터 베이스의 일종. 파일을 저장하는 데이터 베이스

  • 종류

    • ext4: 리눅스 고유 파일 시스템
    • ext3, ext2: ext4의 이전 버전
    • xfs: 실리콘 그래픽스의 저널링 파일 시스템(RHEL7부터 기본 파일시스템)
    • zfs: 솔라리스의 기본 파일 시스템, CoW지원
    • Btrfs: zfs와 비슷한 컨셉으로 개발된
    • ntfs: 윈도우즈 NT 계열에서 사용하는 파일 시스템
    • fat32: FAT 32비트 버전
    • exfat: FAT 32비트 확장 버전

  • 새로운 FS 만드는 법

    • partitioning: fdisk, parted
    • file system: mkfs, fsck or xfs_*
    • mount: mount/unmount. /etc/fstab

  • Partition

    • physical/logical 구획이 있다. 현대 OS는 logical partition을 선호
    • logical portition을 사용하면 물리적 한계를 가진 디스크를 묶어서 보다 큰 용량의 파티션을 만들 수 있다. logical volume(LVM)
    • physical disklabel type
      • DOS 방식: 2TB 제한, fdisk 사용
      • GPT 방식: 무제한 용량, parted/gparted for gdisk 사용
    • Dos partition 종류
      • primary partition(주 파티션) 하나의 디스크에 4개 까지 만들 수 있음
      • extended partition(확장 파티션) 4개 이상 파티션이 필요할 때 primary partition 대신 1개 만들 수 있음. extended partition은 여러 개의 logical drive로 나눌 수 있음.

  • fdisk

    • fdisk -l 현재 FS 리스트를 출력. 요즘 lsblk 사용
    • block device
      • 저장 장치를 이르는 용어
      • /dev/sd SCSI disk, serial type SATA, USB
      • /dev/hd IDE disk
    • fdisk /dev/sda
      • a: 부트 활성 플래그를 지정
      • d: 파티션 삭제
      • l: 알려진 파티션 ID
      • n: 새로운 파티션을 생성
      • p: 현재 파티션 상태를 출력
      • t: 파티션 ID(type)을 변경
      • q: 변경된 상태를 저장하지 않고 종료
      • w: 변경된 상태를 저장하고 종료

  • Make File System

    • 저장 장치는 아주 커다란 종이로 비유하면 파티셔닝은 종이를 몇개로 자르는 작업이다.
    • 파티션 된 종이를 A4와 같은 규격화된 형태로 잘라서 분철하는 것과 유사.
    • 분철을 하면 권수와 페이지 번호가 만들어짐. 인덱싱 가능
    • 분철 방식, 밑줄, 칸의 간격 자물쇄 설치 등에 대한 세부 방식은 file system type에 따름 e.g. ext4, xfs …
    • ext4
      • 저널링 지원
      • 1EB volume 지원
      • 연속된 파일의 접근 및 작은 파일들의 접근이 빠른 장점
      • 저성능 I/O에서 효율이 높다(IoT 기기)
      • 삭제된 파일 복구 가능
    • XFS
      • 저널링 기반 대용량 파일 시스템
      • SGI 실리콘 그래픽스사에서 개발 IRIX 유닉스에서 사용. 후 오픈소스로 이전
      • 2000년 초반에 포팅되어 많은 기간동안 테스트됨. 대부분의 고성능 서버형 리눅스 배포판의 기본 파일 시스템으로 사용됨. Debian, RHEL7
      • Online 상태(mounted)에서 확장이 가능
      • 대용량 파일 처리시 성능이 좋음
      • kernel service(xfs*)가 관리
      • 삭제된 파일을 복구 할 수 없다.
      • 깨진 파일 시스템 복구를 위한 명령어로 xfs_repair를 제공
    • mkfs
      • make file system 명령어
      • mkfs [-t fstype] [option] <device>
      • lsblk 추가된 disk 확인 인식이 안되있으면
      • ls /sys/class/scsi_host/ | while read hostdev ; do echo “- - -” /sys/class/scsi_host/$hostdev/scan ; done 입력 후
      • lsblk 로 블락디바이스 확인후 fdisk /dev/sdb 로 새로운 파티션 생성
      • mkfs -t ext4 /dev/sdb2 혹은 mkfs.ext4 /dev/sdb2 파일 시스템 생성
    • fsck
      • fsck [-ARNV] [options] [-t fstypr] <device>
      • file-system checking 일반적으로 offline 된 FS 체크
      • fsck -y -t ext4 /dev/sdb2 혹은 fsck.ext4 -y /dev/sdb2 로 파일 시스템 체크
      • echo $? 이전 명령어(fsck)의 main() 함수의 리턴값(RC: return code)을 확인하여 검사 결과 확인
    • 스왑 공간
      • swap in/out을 위해 RAM의 최대 2배까지 필요
      • mkswap(스왑 공간 작성:포맷)
        • mkswap -L swapfs2 /dev/sdb1 스왑 파티션에 공간 작성
        • dd if=/dev/zero of=./swapfile1 bs=1024 count=262144 입력후 mkswap ./swapfile1 일반 파일을 스왑 공간으로 사용
      • swapon: 스왑 활성 사용
        • swapon /dev/sdb1, swapon ./swapfile1, swapon -L swapfs2
        • cat /proc/swaps 스왑 공간 확인
        • grep swap /etc/fstab 등록된 스왑 공간 확인
      • swapoff: 스왑 비활성
      • 레이블이 한글로 설정될 경우 영문으로 다시 바꾸어 줘야함

  • mount

    • 파일 시스템의 탑재
    • 마운트는 root directory(/)를 기점으로 시작
    • mount: 마운트를 하는 명령
      • mount [-t fstype] [-o option] [device] <directory>
      • 마운트 해보기
        • mkfs -t ext4 /dev/sdb2
        • mkdir /media/backup
        • mount -t ext4 /dev/sdb2 /media/backup
        • lsblk
      • mount -o ro /dev/sdb2 /media/backup readonly 옵션 적용
      • mount -0 remount, ro /dev/sdb2 리마운트로 옵션 변경 적용 가능
    • umount: 언마운트를 하는 명령
      • umount < directory | device >
      • umount /media/backup, umount /dev/sdb2 디렉토리 디바이스 모두 가능
      • lsblk
    • findmnt: 마운트 리스트 확인
    • ls /dev/disk, ls -l /dev/disk/by-uuid
    • USB memory mount
      • USB 메모리를 장착하면 lsblk에서 장치명이 인식된다.
      • mount 명령으로 인식시키면 된다. FStype을 알고 있어야 된다.
      • blkid /dev/sdc1
    • mount --bind /usr/src/redhat/RPMS /var/www/thml/rpms 디렉터리를 다른 위치의 디렉터리에 붙이는 기능
    • Device is busy!
      • 마운트한 파일시스템에서 프로세스가 실행중인 경우 발생
      • fuser 로 해당 파일 시스템에서 구종 중인 프로세스를 체크/종료
      • fuser -c /media/cdrom 프로세스 체크
      • fuser -ck /media/cdrom 프로세스 체크하여 종료
    • eject /media/cdrom

  • fstab

    • filesystem table
      • /etc/fstab에 위치
      • 부팅시 파일 시스템을 자동 마운트 하기 위한 정보를 담고 있음
      • mount 명령을 단축 시킬 수 있는 정보를 가짐
      • UUID=02a202b2-aee1 /home xfs defaults 0 0
      • device mountpoint fstype options dump,fsck
    • /etc/fstab의 1번 항목에 들어가는 방식
      • /dev.sd* 실제 디바이스 장치로 설정하는 방식. 장치가 순서대로 연결되지 않을 경우 장치명이 바뀔 수 있다!!!
      • LABEL=name 레이블 이름으로 검색하여 장치를 찾아내는 방식
      • UUID=uuid UUID 값으로 검색해서 장치를 찾아내는 방식
    • 4번 항목. 파일 시스템 주요 옵션 mount 옵션과 동일
      • defaults 기본값 사용 - 기본값에선 부팅시 자동으로 마운트 됨
      • noauto 부팅시 자동으로 마운트 하지 않음
      • rw 읽기/쓰기 가능
      • ro 읽기만 가능
      • user 일반 유저도 마운트 가능
      • nouser 일반 유저는 마운트 할 수 없음(슈퍼유저만 가능)
      • noexec 실행 파일의 사용 금지
      • relatime 특정 경우에만 atime 업데이트
      • noatime atime 사용 안함
    • systemd를 사용하는 시스템의 fstab??
    • e2label <device> [new-label] ext4 리눅스 파일 시스템의 레이블을 확인 및 편집
    • tune2fs ext4 파일 시스템의 다양한 설정 조정
      • tune2fs -U <clear|random|time|UUID_value>
      • uuidgen UUID 생성 유틸
    • xfs_admin XFS 파일 시스템의 레이블/UUID 확인 및 편집
    • findfs LABEL=device 레이블을 검색

  • 디렉터리 구조

    • SUS 표준에 근거한 디렉토리 구조

      • /dev 장치 파일들
      • /tmp 임시 파일 저장용 디렉토리
      • /dev/null 널 디바이스
      • /dev/console 시스템 콘솔 장치
      • /boot 부팅을 위한 커널 관련 이미지
      • /bin 기본 실행 명령어 파일들
      • /sbin 수퍼유저용 시스템 관리용 명령어 파일들
      • /etc 시스템 설정 파일 및 관련 스크립트
      • /lib 시스템 라이브러리
      • /lib64 시스템 라이브러리 64bit 전용
      • /usr 응용 프로그램 관련 파일들
        • /usr/bin 응용 프로그램 실행 파일들
        • /usr/sbin 관리자용 응용 프로그램 실행 파일
        • /usr/lib 라이브러리

        • /usr/share 응용프로그램이 공유하는 파일

      • /export /o /o1 외부 저장 장치 사용자들이 만들어서 사용

UNIX

  • Ken Thompson, Ritchie이 게임을 위해 UNIX를 만듬
  • UNIX가 성공하면서 DEC 운영체제가 밀려남
  • 당시 UNIX는 어셈블리어로 작성되어 새로운 CPU가 나오면 거의 새로 작성되어야 했다.
  • Dennis Ritchie는 Brian Kernighan의 도움으로 B 언어를 개량한 C 언어를 개발(1973)
  • UNIX가 C언어로 작성되면서 포팅이 쉬워짐
  • C언어의 부흥으로 어셈블리어의 몰락을 가져옴
  • UNIX는 분열하기 시작
  • BSD UNIX가 탄생 (1975년) 했으나 구심점이 없었다.
  • 기능을 각자 알아서 추가하다 보니 호환성이 사라짐
  • Sysv vs BSD 경쟁 - 명령어도 다름. 운영체제 버전이 다르면 호환이 안됨
  • POSIX가 있어도 더 복잡해진 상황
  • 빌게이츠의 제안으로 SUS - Single UNIX Specification 유닉스 단일 표준안

C Programming Language

  • The C Programming Language (TCPL, K&R)
  • 고급언어이면서도 어셈블리어에 버금가는 성능
  • 낮은 추상화 지원 stdio, File 등
  • 저수준의 하드웨어 조작이 가능
  • 어셈블리어보다 쉬운 언어적 특성
  • ANSI-C(C89): 1989년도 표준
  • C99 - 1999년도 표준
  • C11 : 2011년도 표준

POSIX

  • Portable Oprating system Interface
  • 1988년 POSIX 초안 발표
  • 전기전자공학회(IEEE)의 주도로 system call 표준화 진행
  • POSIX는 API의 문법과 작동에 대한 의미(semantic)만을 담고 있음. 느슨하게 작성됨.
  • 정부 조달의 최소기준

OSF

  • Open Software Foundation 등장
  • 대기업의 반독점을 목적으로 설립된 비영리 단체
  • 1990년 초반 BSD, SVR4, OSF/1 등으로 파편화

X/Open

  • XPG는 manual page의 시초
  • 유닉스 상표권을 소유하게 되나 나중에 Open Group에 넘김
  • SUSv1: XPG 4.2 UNIX95
  • SUSv2: UNIX98 500번
  • SUSv3: SUS 2002 600번 v4는 700

apt-mirror 설치

미러를 하기 위한 패키지를 설치했다.

mirror 설정 파일 편집

/etc/apt/mirror.list 이 파일을 편집한다.

위 파일을 데비안에 apt-mirror를 설치했을 때 생성되는 파일을 편집한 것이다. set base_path /aaa/bbb/ccc 이부분을 편집함으로서 미러링 할때 패키지들이 저장되는 경로를 설정할 수 있다. apt-mirror를 설치할 경우 생성되는 경로는 /var/spool/apt-mirror 이다. 이 디렉토리 아래 var/clean.sh와 var/postmirror.sh 파일이 있는데 base_path를 바꿀 경우 이 파일들도 함께 복사해주면 미러링 시 별다른 알림 없이 진행된다.
deb-amd64 http://update.gooroom.kr/gooroom gooroom-1.0 main 이런식으로 미러링하고자 하는 저장소를 추가한다.
